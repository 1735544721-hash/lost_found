第 2 章 相关理论知识和技术介绍

2.1 SpringBoot框架介绍

Spring Boot 是基于 Spring 生态的轻量化开发框架，强调约定优于配置，通过自动配置与 starter 机制快速集成 Web、数据访问与安全能力，降低基础工程搭建成本。本系统后端采用 Spring Boot 3.2.0 与 Java 17 作为运行基础，结合 spring-boot-starter-web 提供 REST 接口，配合 Spring Security 与 JWT 完成身份认证与权限校验；数据层引入 MyBatis-Plus 3.5.7 统一 CRUD 与分页能力，配合 Hutool 工具库简化日期、加密与文件处理逻辑。控制层、服务层与 Mapper 层形成清晰分层，分别承载失物与招领信息维护、认领申请审核、通知发布与文件上传等业务流程，借助 Knife4j 输出接口文档，保证多端联调与后续迭代的可维护性。

2.2 Vue.js前端框架

Vue.js 是渐进式 JavaScript 前端框架，采用响应式数据驱动与组件化组织方式，可在单页面应用中保持结构清晰与交互流畅。本系统前端基于 Vue 3 与 Vite 构建，配合单文件组件与路由机制实现页面按功能拆分与按角色加载，为失物招领业务提供轻量、可维护的界面基础。

在失物招领系统的界面实现中，前端将失物发布、招领信息展示、分类筛选、认领申请、消息通知与个人中心等模块拆分为独立组件，通过 Vue Router 组织导航路径，确保用户与管理员的操作入口清晰一致。Element Plus 提供表格、表单与弹窗组件，适配物品信息录入、审核与状态更新等交互场景；Axios 统一封装接口请求，与后端 REST 服务保持稳定对接；Pinia 与 Vuex 用于维护登录态、角色权限与检索条件，避免跨页面数据丢失，从而支撑高频查询与审核流程的连续性。

2.3 MyBatis-Plus数据访问框架

MyBatis-Plus 是对 MyBatis 的增强框架，强调“零侵入”与约定式开发，在保留原生 SQL 控制能力的同时，提供通用 CRUD、条件构造器与分页插件，降低数据访问层的模板化代码负担。本系统采用 mybatis-plus-spring-boot3-starter 3.5.7 进行集成，配合 Spring Boot 3.2 的自动配置机制，使实体类与数据库表的映射关系更清晰，便于在业务演化中保持持久层结构稳定。

在实现层面，系统各核心实体（如失物、招领、认领申请、通知与用户）均以 BaseMapper 作为数据访问入口，Service 层在 ServiceImpl 基础上封装通用增删改查与分页检索。业务筛选与状态流转主要通过 LambdaQueryWrapper 组织条件，配合驼峰映射与字段别名配置，减少手写 SQL 的重复与误差，使数据查询与审核逻辑保持一致性与可维护性。

2.4 MySQL数据库

MySQL 是成熟的关系型数据库管理系统，具备事务一致性与索引检索能力，适用于结构化业务数据的持久化管理。本系统使用 MySQL 作为主数据存储，数据库名为 lost_found_db，连接配置包含 UTF-8 字符集与时区设置，满足中文信息展示与时间字段记录的需求。核心表围绕失物信息、招领信息、物品分类、认领申请、通知消息与用户账户展开，形成围绕校园物品流转的结构化数据体系。

2.5 Java语言

Java 是面向对象的通用型开发语言，具备良好的跨平台特性与稳定的生态支持，适合构建中大型 Web 应用。本系统后端基于 Java 17 实现，结合 Spring Boot 的依赖注入与注解机制组织控制层、服务层与数据层，使业务逻辑具备明确的职责划分与可扩展性。实体类用于承载失物、招领、认领申请与通知等领域对象的状态，控制层通过 REST 接口完成前后端交互，服务层则承担审核流转、权限校验与通知触发等核心逻辑，配合 Lombok、JWT 与工具库实现样板代码压缩与安全认证支撑，从而保障系统在功能完整性与维护成本之间取得平衡。
